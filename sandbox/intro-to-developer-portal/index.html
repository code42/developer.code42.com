    Code42 API documentation

    Select APIs > API Reference in the top menu bar to access the Code42 API documentation. The APIs are divided into the following categories:

    • Rules: APIs to manage the rules used to generate alerts. To learn how to use these APIs, see the Security alerts API article.
    • Alerts: APIs to manage the alerts generated by alert rules. To learn how to use these APIs, see the Security alerts API article.
    • Cases: APIs to manage cases.
    • File Events: APIs to search for file events using Forensic Search, and to manage file events in cases. To learn how to use Forensic Search APIs, see the Forensic Search API article.
    • Auth: API to generate authentication tokens. To learn how to use the authentication API, see Authentication below.
    • User: APIs to manage user information, including creating a user profile for use in detection lists, adding risk factors to users, and adding cloud usernames to users.
    • Departing Employee: APIs to manage users in the Departing Employee list. To learn how to use these APIs, see the Detection List Management API article.
    • High Risk Employee: APIs to manage users in the High Risk Employees list. To learn how to use these APIs, see the Detection List Management API article.

    Request URLs

    Following are the request URLs for the APIs shown in the Code42 API Documentation.

    • United States:

      • If you sign in to the Code42 console at https://www.crashplan.com/console, use https://api.us2.code42.com/<resource>.
      • If you sign in to the Code42 console at https://console.us.code42.com/console, use https://api.us.code42.com/<resource>.

    • Ireland: If you sign in to the Code42 console at https://console.ie.code42.com/console, use https://api.ie.code42/<resource>.

    Resources

    Combine the request URL with the resource to get the full URL for the API. The path of the resource is shown for each API listed in the Code42 API Documentation.

    For example, the Auth API documentation shows the resource as /v1/auth/. To run the Auth API, combine the request URL with the resource in the API call:

    curl -X GET -u "username" -H "Accept: application/json" "https://api.us.code42.com/v1/auth/basic?useBody=true"

    Authentication

    Most API calls require an authentication token in the header of the request. To obtain an authentication token, use your Code42 administrator credentials to submit a GET request to the /v1/auth/ API. (For details, see the Auth API documentation.)

    In the following example, replace <Code42Username> with a Code42 administrator username, and replace <RequestURL> with the request URL of your Code42 cloud instance:

    curl -X GET -u "<Code42Username>" -H "Accept: application/json" "https://<RequestURL>/v1/auth/basic?useBody=true"

    If your organization uses two-factor authentication for local users, you must also include a totp-auth header value containing the Time-based One-Time Password (TOTP) supplied by the Google Authenticator mobile app. The example below includes a TOTP value of 424242.

    curl -X GET -u "<Code42Username>" -H "totp-auth: 424242" "Accept: application/json" "https://<RequestURL>/v1/auth/basic?useBody=true"

    A successful request returns an authentication token. For example:

    {"bearerToken": "eyJjdHkiO_bxYJOOn28y...5HGtGHgJzHVCE8zfy1qRBf_rhchA"}

    Token considerations

    • Authentication tokens expire after 30 minutes.
    • The administrator user whose Code42 username is used in the authentication API request must have the Customer Cloud Admin role.

    Get a tenant ID

    Some APIs require that you provide the unique ID of your organization (or tenant) in the Code42 cloud. To obtain the tenant ID, use your Code42 administrator credentials to submit a GET request to the /v1/customer API.

    In the following example, replace <AuthToken> with the authentication token and replace <RequestURL> with the request URL of your Code42 cloud instance:

    curl -vvv -X GET -H "Authorization: Bearer <AuthToken>" 'https://<RequestURL>/v1/customer'

    A successful response returns the tenantUid:

    Get userUIDs

    Some APIs require that you provide a user's UID. Each user in Code42 is uniquely identified by a userUID value.

    You can locate userUIDs directly in the Code42 console:

    • Export a list of Code42 users to a CSV file. The file includes each user's userUID.
    • View userUIDs in the User Backup report and Device Status report.

    You can also submit a GET request to the /v1/User API to get a userUID (see the User API documentation).

    In the following example, replace <RequestURL> with the request URL of your Code42 cloud instance, replace <Code42Username> with the Code42 username of the user whose userUID value you want to view, and replace <AuthToken> with the authentication token:

    curl -X -GET 'https://<RequestURL>/v1/users?username=<Code42Username>' -H 'Authorization: Bearer <AuthToken>'

    API Considerations

    • For assistance with using the Code42 API, contact your Customer Success Manager (CSM) to engage the Code42 Professional Services team. Or, post your question to the Code42 community to get advice from fellow Code42 administrators.

    • API resources themselves only work for you under these conditions:

      • You have a product plan that includes access to the Code42 API.
      • Your credentials rely on local authentication. SSO or authentication through any third-party provider will not work.
      • Your role provides permission to access the data necessary to a given API resource. If your API calls fail because you do not have permission to use them, you will see reply messages like these: "HTTP 401 Unauthorized", "HTTP 401 Could not authenticate user", "Your Code42 product plan does not permit use of the Code42 API".
      • The examples in the articles here use curl. For other tools that you can use, see Tools for interacting with the Code42 API.
